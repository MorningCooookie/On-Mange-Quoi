## VISION & MISSION

### Why We Exist
Food scandals (cadmium, mercury, pesticides) create fear and decision fatigue for families. We solve it.

**Mission:** Take the cognitive load OFF families. One service = safe menus + recipes + shopping list, all coordinated.

### What We're NOT
- ❌ A multi-profile "family sharing" feature (users don't pay for multiple accounts)
- ❌ A recipe platform competing with 1000s of recipe sites
- ❌ A shopping discount aggregator
- ❌ A generic meal planner

### What We ARE
**On Mange Quoi = Meal Planning That Cares About Your Body**

A freemium meal planning assistant that:
1. **Adapts** weekly menus to YOUR dietary needs (allergies, restrictions, preferences)
2. **Verifies** safety against ANSES/EFSA food standards
3. **Generates** recipes for your meals
4. **Organizes** shopping lists by category + allergen alerts
5. **Optimizes** for budget, time, nutrition, and waste (premium features)

## TECHNICAL ARCHITECTURE

### Database Schema
```
Profiles table (existing):
  - id, user_id, name, created_at

Preferences table (new):
  - id, profile_id, user_id
  - allergies[] (peanuts, shellfish, dairy, gluten, eggs, soy, fish, tree nuts)
  - restrictions[] (vegan, vegetarian, kosher, halal, low-sodium, low-sugar)
  - dislikes[] (free text, up to 5 items)
  - created_at, updated_at

Premium_features table (future):
  - id, user_id, feature_name, enabled, created_at
```

### API Endpoints
```
GET /api/menu?profile_id=xxx
  → Returns filtered menu for profile (with preferences applied)

POST /api/preferences
  → Save dietary preferences for a profile

GET /api/preferences?profile_id=xxx
  → Load preferences for a profile

POST /api/recipes/generate
  → Generate recipe via Claude (filtered by preferences)

GET /api/shopping-list?profile_id=xxx
  → Shopping list organized by category
```

### Frontend Structure
```
Components:
  - PreferenceManager (modal to edit preferences)
  - MenuView (filtered menu display)
  - ProfileSelector (switch between profiles)
  - ShoppingList (organized by category)
  - Premium Upsell (contextual messaging)

Pages:
  - Landing (sign up)
  - Onboarding (set preferences)
  - Dashboard (home page with menus + recipes + shopping)
  - Settings (edit preferences, manage profiles)
```

## REMEMBER (Core Principles)

1. **User-first, not feature-first**
   - Don't add features because they're cool
   - Add features because users ask for them or will pay for them

2. **Freemium is trust-building, not monetization**
   - Free tier solves the core problem (food safety)
   - Premium is "advanced meal planning" (for power users only)
   - If users don't see value in free tier, they won't convert to premium

3. **Family is coordination, not duplication**
   - One dinner, multiple diets
   - Not "multiple accounts," not "family plans with child accounts"
   - Real value: "I plan once, everyone eats differently"

4. **Food safety is your moat**
   - Verify against ANSES/EFSA (only you do this)
   - Build trust with regulatory compliance
   - Use it as a differentiator (not just "meal planner")

5. **Ship fast, iterate on feedback**
   - Week 1-2: Ship core (personalized menus)
   - Week 3-4: Ship premium (budget + planning)
   - Week 5+: Ship what users ask for
   - Don't wait for perfection
